Project Structure Overview

In this transcript, I share my approach to organizing project folders for a boat ramp project. I start by explaining that at the root of my project folder, I keep the final plans along with the cost estimate. I've created several subfolders for different types of materials, including CAD files, drone pictures, and survey information.

I emphasize the importance of having separate files for different aspects of the project. For example, I create distinct CAD models for each section of the plan, such as layout, details, and existing conditions. This way, I can manage what gets displayed without having to deal with multiple layers across various pages. By chunking the information into smaller models, I streamline the plotting process and minimize the need to turn off layers individually.

I also discuss my setup for the details page, where I draw elements to scale. I make a copy of my page in one-to-one model space, allowing me to create different viewports at varying scales. This setup ensures that dimensions and details remain accurate and give the flexibility of having multiple scales on the same page.

Finally, I summarize how I group related files, such as existing and proposed conditions, to keep my project organized. This structured approach helps me effectively manage the creation of the ramp plan set and ensures I can easy find and modify files in the future if needed.
